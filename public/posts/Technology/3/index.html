RTSP & Packet Proxy

프로토콜

RTSP 프로토콜을 따르는 IP Camera는 보통 아래 3개의 프로토콜을 이용해 미디어 접속 상태를 관리하고, 데이터를 전송한다.

  1. RTSP (Real Time Streaming Protocol)
    • 클라이언트와 서버의 연결 상태를 유지. 스트리밍의 시작, 중지, 일시정지 등의 제어에 관여
    • 실제 미디어 데이터의 정보는 포함하지 않음
    • TCP 통신을 사용함
  1. RTP (Real time Transport Protocol)
    • 실시간으로 미디어 데이터를 전송하는 프로토콜
    • 패킷의 순서를 보장하지 않음. 시퀀스 번호를 포함하여 수신자가 재조립하여 재생
    • UDP, TCP 모두 가능하지만 일반적으로 UDP 통신을 사용함
  1. RTCP (RTP Control Protocol)
    • RTP 스트림의 품질 모니터링 및 제어를 위한 프로토콜
    • 네트워크 상태 모니터링, 대역폭 사용률, 패킷 손실, 지연 시간 등의 정보를 주고 받으며 피드백을 제공

RTP Packet 의 구조

  1. Header (12 bytes)
    • version: 2 bits
      • RTP 프로토콜의 버전
    • padding: 1 bit
    • CSRC Count: 4bits
    • Marker: 1 bit
      • 이벤트를 표시. (프레임의 끝, 중요한 데이터의 시작 등)
    • Payload Type: 7 bits
      • 오디오, 비디오 코덱 유형
    • Sequence Number: 16 bits
      • 패킷의 순서를 tracking하기 위해 사용됨
    • Timestamp: 32 bits
      • 재생 타이밍 맞추기 위한 용도(?)의 timestamp
      • 평소에는 바꿔도 영상에는 특별한 문제가 없으나 각종 장애 상황으로 영상 품질이 매우 좋지 못할 때 가끔 내부적으로 timestamp를 참고해서 조정하는 듯 합니다. (확실하지 않음)
    • SSRC: 32 bits
      • stream을 구별하기 위한 고유 식별자
    • CSRC (optional)
      • 원본 데이터 소스의 식별자
  1. Payload
    • 미디어 데이터. 코덱, 응용 프로그램에 따라 구조가 약간씩 다름

Packet Proxy

Linux / Ubuntu 환경에서는 ip-tables를 이용해 packet의 proxy를 둘 수 있다.

  • iptc rule & nfqueue
    import iptc
import random
from netfilterqueue import NetfilterQueue, Packet

def packet_filter(packet: Packet):
    # you may modify packet here
    if random.random() < 0.5:
        packet.accept()
    else:
        packet.drop()

# 아래 조건과 같은 rule을 iptable에 추가
# src = dst = localhost# protocol = udp
# destination port = 8000
# target = netfilterqueue #1

chain = iptc.Chain(iptc.Table(iptc.Table.FILTER), 'INPUT')

rule = iptc.Rule()
rule.src = '127.0.0.1'
rule.dst = '127.0.0.1'
rule.protocol = 'udp'
match_ = rule.create_match('udp')
match_.dport = '8000'

target = iptc.Target(rule, 'NFQUEUE')
target.set_parameter('queue-num', '1')
rule.target = target
chain.insert_rule(rule)

# bind nfqueue
queue = NetfilterQueue()
queue.bind(1, packet_filter)

try:
	queue.run()
except:
	print('nfqueue terminated')

# delete rule
chain.delete_rule(rule)

# unbind nfqueue
queue.unbind()

RTP Packet Manipulation

scapy를 이용해 packet을 편하게(?) 뜯어보고, 변조하고, 다시 전송할 수 있다.

from scapy.all import IP, UDP, L3RawSocket, Raw, conf, send
from netfilterqueue import Packet

conf.L3socket = L3RawSocket

def packet_filter(packet: Packet):
    """peek timestamp, sequence#, marker"""
    scapy_packet = IP(packet.get_payload())
    udp_payload = scapy_packet[UDP].payload.load
    rtp_header = bytearray(udp_payload[:12])
    timestamp = int.from_bytes(rtp_header[4:8], byteorder='big')
    sequence_number = (rtp_header[2] << 8) | rtp_header[3]
    marker = (rtp_header[1] & 0x80) >> 7
    print(timestamp, sequence_number, marker)

def send_packet(packet: Packet):
    """send a packet"""
    scapy_packet = IP(packet.get_payload())
    send(scapy_packet, verbose=False)
