Flux Architecture & Redux

Flux Architecture

Facebook이 만든 단방향 데이터 흐름을 갖는 application architecture로 상태 관리의 단순화, 데이터 흐름 관리에 용이

Action

  • 데이터의 흐름에 변화를 주는 모든 이벤트
  • type과 payload로 구성

Dispatcher

  • action을 store로 보내는 중앙 허브

Store

  • application의 상태를 저장하고 업데이트

View

  • 사용자가 보는 ui, store에서 상태가 변경되면 자동으로 업데이트

Redux

ReduDan Abramov가 flux에서 영감을 받아 만든 상태 관리 라이브러리

  1. Single source of truth
    1. 모든 상태를 단일한 store에 저장됨
    1. 중앙 집중식 상태 관리, 데이터 일관성 확보에 용이
  1. State is read-only
    1. 정해진 상황과 규칙에 따라서만 상태의 변화가 발생하며 외부에서 임의로 상태를 변경할 수 없음
  1. Changes are made with pure functions
    1. pure function: 입력값을 바꾸지 않고 새로운 객체를 return하는 함수
    1. 그러한 pure function (reducer)를 통해 상태의 변화가 발생

Redux thunk

  • Redux에서 비동기 작업을 처리할 수 있게 해주는 미들웨어
    • action과 reducer 사이에서 동작
  • 비동기 action을 작성하고 dispatch할 수 있는 기능을 제공
  • 아래는 chat-gpt가 뱉은 예제 코드로 아마도(?) 옳은 코드일 것
    • 느낌만 익히고 넘어가자
import { createStore, applyMiddleware } from 'redux';
import thunk from 'redux-thunk';
import rootReducer from './reducers';

// Redux 스토어 설정에 Thunk 미들웨어 적용
const store = createStore(
  rootReducer, 
  applyMiddleware(thunk)
);

const fetchItems = () => {
  return async (dispatch, getState) => {
    dispatch({ type: 'FETCH_ITEMS_REQUEST' });

    try {
      const response = await fetch('/api/items');
      const data = await response.json();

      dispatch({
        type: 'FETCH_ITEMS_SUCCESS',
        payload: data
      });
    } catch (error) {
      dispatch({
        type: 'FETCH_ITEMS_FAILURE',
        error: error.message
      });
    }
  };
};

store.dispatch(fetchItems());
